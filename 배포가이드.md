# 급여 계산 시스템 배포 가이드

## 전체 흐름
GitHub에 코드 올리기 → Supabase DB 만들기 → Vercel 배포 → 완성!

---

## STEP 1. GitHub에 코드 올리기

1. https://github.com/new 접속
2. Repository name: `comma-payroll` 입력
3. Private 선택 → Create repository
4. "uploading an existing file" 클릭
5. 이 폴더 안의 파일들을 폴더 구조 그대로 업로드
6. Commit changes 클릭

### 폴더 구조
```
comma-payroll/
├── pages/
│   ├── _app.js
│   ├── index.js          ← 급여 계산 메인
│   ├── payroll-manager.js ← 관리자 현황 페이지
│   └── api/
│       └── save.js
├── lib/
│   └── supabase.js
├── package.json
└── next.config.js
```

---

## STEP 2. Supabase DB 설정

1. https://supabase.com → 기존 프로젝트 사용 or 새 프로젝트 생성
2. SQL Editor → `payroll-schema.sql` 내용 붙여넣기 → Run
3. Settings → API에서 URL과 anon key 복사

---

## STEP 3. Vercel 배포

1. https://vercel.com → Add New → Project
2. comma-payroll 저장소 선택 → Import
3. Environment Variables 입력:

| 이름 | 값 |
|------|-----|
| NEXT_PUBLIC_SUPABASE_URL | Supabase Project URL |
| NEXT_PUBLIC_SUPABASE_ANON_KEY | Supabase anon key |
| NEXT_PUBLIC_MANAGER_PASSWORD | 관리자 마스터 비밀번호 |

4. Deploy 클릭!

---

## 접속 URL

- 급여 계산: `https://your-url.vercel.app`
- 급여 현황 관리: `https://your-url.vercel.app/payroll-manager`

---

## 지점별 비밀번호 변경

`pages/index.js` 맨 위 BRANCHES 배열에서 각 지점의 password를 수정하세요:

```javascript
const BRANCHES = [
  { id: 'gidc',   name: '광명GIDC점',  password: '원하는비밀번호' },
  { id: 'ingye',  name: '인계점',       password: '원하는비밀번호' },
  ...
]
```

---

## 급여 계산 방식

- **기본수당**: 근무시간 × 시급
- **연장수당**: 연장시간 × 시급 × 1.5
- **야간수당**: 야간시간 × 시급 × 0.5 (22:00~06:00)
- **휴일근로**: 휴일시간 × 시급 × 1.5
- **휴일연장**: 휴일연장시간 × 시급 × 2.0
- **휴일야간**: 휴일야간시간 × 시급 × 0.5
- **주휴수당**: 주 15시간 이상 근무 시 지급
